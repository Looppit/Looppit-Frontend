# 규칙 파일 작성 가이드

## 원칙

### 1. 간결함 유지

- 개별 Rule은 **50줄 미만**으로 유지
- 핵심 내용만 포함하고 불필요한 설명 제거

### 2. 큰 개념은 나누기

- 여러 Rule로 나눠서 조합
- 각 Rule은 단일 책임 원칙을 따름
- 관련 규칙은 상호 참조로 연결

### 3. 구체적인 예시 제공

- 필요시 참조 파일 사용 (예: `@filename.ts`)

### 4. 명확하게 작성

- 모호한 지침은 피하고 내부 문서 작성처럼 명확하게
- 체크리스트로 검증 가능하게 작성
- "해야 한다", "하지 않는다" 등 명확한 표현 사용

## 구조 템플릿

```markdown
# 규칙 제목

## 개요

- 핵심 원칙 요약

## 세부 지침

- 구체적인 규칙 설명

## 체크리스트

- [ ] 검증 항목
```

## 참조 파일 표기법

- `@/filename.ts` - 파일 참조
- `@/path/to/file.ts` - 경로 포함 파일 참조
- `.github/pull_request_template.md` - 특수 경로 파일 참조

## 규칙 간 상호 참조

- 다른 규칙 참조 시: `(naming-conventions.md 참고)`
- 관련 규칙 나열: `(code-style.md, naming-conventions.md 참고)`
